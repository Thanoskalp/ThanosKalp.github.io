Memory Game

A simple memory game with cards playable in 3 difficulties

This was the first project I had to create from zero in Java. I worked with one of my best friends and excellent student Kostas Alexandris. The user interface (UI), but that wasn't the point of the project. The main point was to use correctly all the object-oriented principles(encapsulation, abstraction, inheritance, and polymorphism), as well as having small classes and keeping the code as clean as possible for that level.

About the game

Launching the game you have to insert your name and choose a difficulty. The 3 difficulties are Easy (4x3), Medium (5x3), and Hard(6x6). The game is played in 2 turns, the player goes first and the computer second. Whenever you find a matching pair you get 10 points. Whoever gets the most points wins. There is also a 3-minute timer. If the timer ends and the pairs were not found, the player with the highest score wins. All the data (Name and score) are stored in an XML.

FPS Vr Game

My first project using C++ scripting in UE4

After being accepted by one of my instructors in his lab, he provided me with a PC with a GTX 1060 6gb and HTC Vive. I was asked to create something using C++ for scripting and no blueprints for the most part. I created a simple first-person character who just shoots projectiles with some simple physics. I imported the library needed to bind the Vive controllers for movement and shooting. Then I added some simple models(like soda cans or bottles) and made them disappear when shot. After one point, I realized that I didn't want to just create functionalities in C++, so I started editing a simple terrain(adding models, sculpting, etc.). I created a new world, added some new materials, lights, models, and textures. As you can see in the picture I only have a small park, some roads, and some buildings. I am willing to take it a step further, considering the idea of making the game for 2 players, so we can play on our breaks at the lab 1 VS 1.
